# TASKS

このファイルは、AI エージェントへ実装指示を渡すためのタスク一覧です。

## 運用ルール

- タスクはチェックボックス形式で記述する
- 1 行目のタスクは、できるだけ機能要求を短く表す
- まだ着手しない検討中の論点は `NOTES.md` に書いてよい
- 実装開始前に、対象タスクを詳細タスクへ分解する
- 詳細タスクは、このファイル内へ追記して管理する
- 実装は TDD で進める
- 実装前に `make check` を実行し、現状の問題有無を確認する
- 先にテストを追加し、失敗を確認してから実装する
- 実装後に再度 `make check` を実行し、問題がないことを確認する
- 完了したタスクは `- [x]` に更新する
- 一時的なメモではなく、完了状態が分かる粒度で残す

## 検討中

- [ ] サンプル検討: 課金転換戦略変更の前提条件を整理する
  - メモは [NOTES.md](./NOTES.md) に残す
  - 実装着手が決まったら未着手タスクへ移す

## 未着手タスク

- [ ] サンプルタスク: 課金転換戦略をデータロック（30日制限）に変更する
  - [ ] 対象仕様と影響範囲を確認する
  - [ ] 必要な詳細タスクへ分解する
  - [ ] `make check` を実行して現状を確認する
  - [ ] 失敗するテストを先に追加する
  - [ ] 実装を行う
  - [ ] `make check` を再実行して成功を確認する
  - [ ] 必要なら `docs` を更新する

## 詳細タスク分解テンプレート

- [ ] 親タスク名
  - [ ] 仕様確認
  - [ ] 影響範囲確認
  - [ ] テスト観点洗い出し
  - [ ] `make check` 実行
  - [ ] 失敗テスト追加
  - [ ] 実装
  - [ ] `make check` 再実行
  - [ ] `docs` 更新要否確認

## タスク化前メモの流れ

1. 検討段階の論点や要確認事項を [NOTES.md](./NOTES.md) に書く
2. 実装することが決まったら、このファイルの `検討中` へ短く追加する
3. 着手時に詳細タスクへ分解して `未着手タスク` へ移す
