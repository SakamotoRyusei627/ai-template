# 0001-documentation-structure

## ステータス

- 採用

## 日付

- 2026-06-14

## 背景

- AI エージェントが作業のたびにリポジトリ全体を読み込むと、トークン効率が悪くなる
- 作業内容ごとに必要な文書だけ読める入口構造が必要だった

## 決定

- `docs/index.md` を入口とし、カテゴリごとの `index.md` を経由して詳細文書へ進む構造を採用する
- 各カテゴリ配下の詳細文書は、必要になったときだけ読む前提にする
- エージェント別入口ファイルは維持しつつ、共通指示は `AGENT_GUIDE.md` に集約する

## 理由

- 入口を固定すると、読み始める順序を統一できる
- カテゴリ単位で分離すると、無関係な領域を読まずに済む
- 詳細文書を小さく保ちやすく、更新対象も絞りやすい

## 代替案

- 単一の大きな `README` に集約する案
- 採用しなかった理由:
  - 情報量が増えるほど入口が重くなる
  - 無関係な情報まで毎回読むことになりやすい

## 影響

- 新規文書はカテゴリ構造に従って追加する必要がある
- 入口、カテゴリ索引、詳細文書の役割分担を保つ必要がある
- 関連 `docs` を相互リンクで結び、必要最小限で読めるように保つ
- `AGENTS.md` と `CLAUDE.md` の差分管理は最小化し、共通運用ルールは一箇所で更新する
